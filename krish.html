Project Overview

The City Cab and Taxi Fare Optimization System is designed to enhance transportation efficiency in Gurugram. It focuses on optimizing routes and calculating fares dynamically, taking into account factors like distance, traffic, and demand. The system is based on a custom graph representation of Gurugram's key sectors and landmarks.

Graph Representation

The city of Gurugram is represented as a graph:

  • Nodes: Key locations such as Sector 14, Cyber City, Huda City Centre, and more.
  • Edges: Roads connecting these locations.
  • Edge Weights: Distances or estimated travel times between nodes.

Example:

  • Sector 14 ↔ Sector 15 (Distance: 5 km)
  • Sector 15 ↔ Cyber City (Distance: 8 km)
  • Cyber City ↔ Udyog Vihar (Distance: 3 km)

Algorithm: Dijkstra's Algorithm

Dijkstra's Algorithm is used to find the shortest path between pickup and drop-off points. This ensures optimal routes that reduce both travel time and fare.

Fare Calculation

Fares are calculated dynamically based on the following components:

  • Base Fare: Fixed cost for booking a ride.
  • Distance Fare: Calculated as Distance × Rate per km.
  • Surge Pricing: Dynamic multiplier during peak hours or high-demand areas.
  • Add-ons: Waiting time charges and toll fees (if applicable).

Example:

  • Base Fare: ₹50
  • Distance Fare (8 km × ₹10/km): ₹80
  • Surge Pricing (1.5×): ₹130 × 1.5 = ₹195
  • Total Fare: ₹195

Auto, Taxi, and Shared Taxi Fare Models

Auto Fare Calculation:

  • Base Fare: ₹40
  • Per km Fare: ₹8/km
  • Waiting Time: ₹2/min
  • Surge Pricing: 1.3× during peak hours

Taxi Fare Calculation:

  • Base Fare: ₹50
  • Per km Fare: ₹12/km
  • Waiting Time: ₹3/min
  • Surge Pricing: 1.5× during peak hours

Shared Taxi Fare Calculation:

  • Base Fare: ₹30
  • Per km Fare: ₹6/km
  • Shared Ride Discount: 20% off

Advanced Features

The system can simulate traffic patterns by dynamically adjusting edge weights based on time of day and integrate real-time data to improve accuracy further.

Algorithm: Dijkstra's Algorithm

Dijkstra's Algorithm is a fundamental part of the system, used to determine the shortest path between the user's pickup and drop-off locations. By representing Gurugram's city map as a graph, where nodes are key locations and edges are roads with weights representing distances or travel times, the algorithm identifies the most efficient route. This not only reduces travel time but also minimizes the fare by avoiding unnecessary detours or longer routes.

Algorithm: String Matching for Coupon Code Validation

To provide a seamless and user-friendly experience, a String Matching Algorithm is implemented to validate coupon codes entered by users. When a user inputs a coupon code (e.g., DAA10), the algorithm efficiently compares it against a predefined list of valid codes using techniques like the Knuth-Morris-Pratt (KMP) or a Naive String Matching approach. If a match is found, the corresponding discount is applied to the calculated fare.

This algorithm ensures quick and accurate validation of coupon codes, enabling dynamic fare adjustments while enhancing user satisfaction. For instance, entering the code DAA10 may provide a 10% discount on the total fare.
